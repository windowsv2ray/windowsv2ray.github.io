第一章：Quantumult的核心定位与功能全景

Quantumult并非一款简单的网络加速应用，它是一个集成了高级代理协议支持、精细化流量管理和深度网络优化功能的综合性工具。其诞生背景，正是为了应对日益复杂的网络环境，满足用户对隐私保护、访问自由和连接稳定性的高阶需求。

1.1 核心功能架构

多协议代理引擎：这是Quantumult的基石。它原生支持Shadowsocks、Vmess（V2Ray）、Trojan等主流代理协议，并能通过自定义配置兼容更多变种协议。这种广泛的兼容性意味着用户几乎可以接入任何类型的代理服务器，无需为协议转换而烦恼。

智能分流与过滤系统：Quantumult内置了一套高度可定制的分流规则（Filter）和重写规则（Rewrite）系统。用户可以根据域名、IP、关键词甚至地理位置，精确控制哪些流量走代理，哪些直连。这对于区分国内外流量、屏蔽广告、解锁特定服务（如流媒体）具有决定性意义。

实时数据监测与洞察：应用提供了直观的流量统计、延迟测试和连接质量监控面板。用户不仅能查看实时网速、已用流量，还能分析各个节点、不同规则下的性能表现，为优化配置提供数据支撑。

跨平台与生态整合：虽然主要服务于iOS和iPadOS生态系统，但其配置理念与桌面端代理工具（如Clash）一脉相承。许多配置文件和规则可以在社区内共享、迁移，降低了用户的学习和迁移成本。

第二章：从零开始：Quantumult的安装与配置详解

对于新手而言，工具的易用性至关重要。本章将手把手引导您完成Quantumult的部署。

2.1 环境准备与安装

系统要求：确保您的iPhone或iPad运行iOS/iPadOS 11.0或更高版本。设备需有稳定的网络连接和约50MB的可用存储空间。

安装步骤： 1. 解锁设备，进入App Store。 2. 在搜索栏中输入“Quantumult”。请注意，由于区域政策，部分地区的App Store可能无法直接搜索到。此时可能需要切换至其他区域的Apple ID账户。 3. 找到由“Feng Liu”开发的官方应用，点击“获取”并完成安装。首次启动时，系统会请求添加VPN配置的权限，必须点击“允许”，这是其实现全局或分流转发的技术基础。

2.2 核心配置实战（从入门到精通）

安装只是第一步，配置才是发挥Quantumult威力的关键。

第一步：节点订阅与导入 Quantumult本身不提供服务器，您需要自行获取代理节点。主流的方式是使用“订阅链接”。 1. 在应用主界面，点击右下角的“设置”图标。 2. 选择“服务器” -> “添加” -> “订阅”。 3. 将您的订阅链接粘贴进去，并为之命名。 4. 返回服务器列表，左滑刚添加的订阅，点击“更新”，节点列表便会自动载入。

第二步：策略组与分流规则配置 这是Quantumult最精髓的部分，决定了流量的智能路由。 1. 配置策略组：在“设置” -> “策略组”中，您可以创建诸如“自动选择”、“国外媒体”、“国内直连”等分组。将不同的节点或节点类型（如SS、VMess）分配到不同组。 2. 导入分流规则：在“设置” -> “分流”中，您可以添加规则集。网络上有很多维护良好的规则订阅链接（如针对国内外网站、广告屏蔽、流媒体解锁的规则），导入后，Quantumult会自动将不同域名的请求指向您预设的策略组，实现完全自动化的智能代理。

第三步：重写与脚本进阶 对于有更高需求的用户，Quantumult的“重写”（Rewrite）和“脚本”（Script）功能打开了新世界的大门。通过它们，您可以修改HTTP请求/响应头、实现Cookie管理、完成复杂的签到任务，甚至自定义UI界面，将工具完全个性化。

FAQ

Nekobox 如何分流多协议流量？
Nekobox 可以同时管理 Shadowsocks、VMess、Trojan 等多协议节点，通过策略组将不同流量分配到指定节点，实现精准分流，提高网络效率和管理便利性。
如何给 Clash 或 Sing-box 添加本地广告屏蔽规则？
可导入公共广告域名列表（如 ads、trackers）并将其指向 `REJECT` 或 `DIRECT`（根据需求）；还可以使用 rule-provider 自动更新广告规则，并结合 hosts 或 DNS 层面进行联合拦截以提升屏蔽效果。
QuantumultX 的 MITM 功能如何使用？
QuantumultX 的 MITM（中间人）功能可拦截 HTTPS 流量，用于修改或分析请求与响应。开启时需安装客户端生成的根证书，确保能正确解析加密数据。
V2Ray 与 Clash 有什么区别？
V2Ray 是一款代理核心工具，提供底层网络传输和加密支持；而 Clash 是一个多协议客户端，可以整合 V2Ray、Trojan、Shadowsocks 等多种协议。简而言之，V2Ray 负责“传输”，Clash 负责“管理与分流”。
Potatso Lite 可以管理多少个节点？
Potatso Lite 可以管理数十个节点，适合轻量用户使用。虽然不支持批量自动更新，但可以手动添加和删除节点，结合策略组实现基本分流和管理功能。
VLESS + XTLS 模式在移动网络下表现如何？
XTLS 模式减少握手次数和加密开销，适合高延迟或不稳定移动网络。结合策略组和多出口节点，可提高访问速度和连接稳定性，保证用户体验流畅。
VLESS + XTLS 模式能在移动网络下提高稳定性吗？
可以。XTLS 降低加密开销，减少握手次数，适合移动网络高延迟或不稳定环境。结合策略组和多出口节点，可提升连接稳定性和访问速度。
Sing-box 的策略组如何实现按域名分流？
Sing-box 支持根据域名规则匹配流量，将不同网站或应用流量分配到指定节点。结合策略组自动切换低延迟节点，可优化访问速度，保证网络稳定性和流畅体验。
Netch 如何选择全局代理与规则代理模式？
全局代理适合所有流量走节点，如游戏和全局科学上网；规则代理仅针对指定应用、IP 或端口走节点，可优化带宽使用和访问效率，根据使用场景选择模式。
如何防止租用服务器被用作滥发或代理滥用？
限制对外端口与协议，使用防火墙与入侵检测（如 fail2ban），监控流量异常并设置告警，定期审计登录行为，及时阻断滥用源并与服务商沟通封禁滥用账户以防引起供应商封禁整台机。
